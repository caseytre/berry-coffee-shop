# Project Retrospective

**Project Name:** Berry Coffee Shop Website
**Completed by:** Casey Trejo
**Date:** July 2026

---

## What Went Well

Setting up the repository structure early made the rest of the project easier to follow. Having the scope document done first meant I always knew what I was building and could avoid adding things that weren't planned. The two-page structure kept the project focused, and the shared stylesheet made styling both pages consistent without a lot of extra work.

---

## What Went Wrong

The biggest challenge was getting comfortable with Git and GitHub early on. Pushing files, resolving conflicts, and keeping the repo organized took more time than expected and caused some confusion along the way. If this were a real project with a deadline, those delays could have been a problem.

---

## What I Would Do Differently

I would set up the GitHub repo and folder structure before writing a single line of code or documentation. Getting that foundation right from the start would have saved time and avoided a lot of back and forth. I would also write the README earlier in the process instead of leaving it to the end.
